Features

Data types

System


* Data advertising system

eGenVar can be used in the same way regardless of whether the data is big or small, private or public , images or text. eGenVar, how was it generated, how it is liked to rest of the data  and where they are located. As the actual files do not reside in the system, there is less concern for security

* The link between LIMS and research data

Can be used to link between your LIMS (Laboratory Information Management System) and the data from research conducted using the samples.

* All in one package

The server, data-management system, user management system, web interface, command line clients, back up management and many more feature comes as a single package. You do not need anything else (except Java and a web browser) to installing or start using the system. As the EGDMS has embedded application server (Glassfish) and a database management system (Derby/JavaDB) you can configure everything from one interface. All components occupy one process in your system (there will be many threads, but one process ID) and the communication between the components are programmatic (not across a network) 

* Connect using many popular programmatic  languages

The webservices can be consumed from Java, Python, R and other programming languages and scripts.

* Meta-data webinterface 

Manage data within a group or an institute with less effort. Once meta-data is recorded the web interface will be generated automatically. This means you get a bonus interface that can be accessed from any web browser (firefox recommended) without coding any HTML.  

* Intelligent  server

Too much load, memory getting low or requires a restart, the server emails you about it. Then take necessary action if you are on holiday and did not respond back.  

* Transform existing data arrangements to data-management

Are you a person using the folder names and subfolders to arrange your data ? Just point eGenVar to where you have your data and it will automatically extract meta-data and create a data management system for you.

* Relocation help

Are you deploying a new data server or moving to cloud storage? Record everything in EGDMS before the transition (if you have not already using EGDMS) and run the sync function when the transition is complete. Then EGDMS will tell you where you old data currently is. 

* Detect duplicates

Big data can become huge if you keep many copies of it. It will cost money and course ambiguity. eGenVar system will provide a report of file level and folder level duplicates. It will link all copies to a single file entry using file checksum. These could be used to detect data duplicates,
